AI-Driven Predictive Autoscaling Engine (AIOps)

KEDA Amazon EKS Python ML Prometheus AIOps

Project Overview

Traditional Kubernetes autoscaling architectures rely heavily on reactive resource metrics (such as Horizontal Pod Autoscaling evaluating sustained CPU or Memory bounds over a fixed window). While effective for gradual traffic shifts, reactive scaling fails to counter sudden, high-velocity surges. By the time threshold metrics trigger a scale-up operation, the target infrastructure is already experiencing bottleneck-driven lag. This project addresses the metric-lag problem by replacing reactive triggers with proactive, intelligence-driven time-series predictions.

Architecture & Technical Execution

The solution pairs advanced data sourcing with native Kubernetes event loops:

  • Telemetry Engine: Seamlessly pulls live traffic metrics directly from Prometheus telemetry streams into a dedicated Python environment.
  • Predictive Forecast Layer: Utilizes an unsupervised Machine Learning model running glue-code components to continuously process time-series metrics and project future traffic trends before they arrive.
  • Dynamic Event Injection: Feeds forecasted metrics into KEDA (Kubernetes Event-driven Autoscaling) via dynamic triggers. This forces preemptive node group expansions on Amazon EKS ahead of anticipated traffic surges.

Measured Impact

  • 40% Latency Mitigation: Preemptive nodes spin up before load execution vectors hit, reducing cluster response latency by 40%.
  • Neutralized Metric Lag: Substituted traditional threshold alerts with predictive anomaly signatures, neutralizing metric-lag dropouts entirely.
  • Infrastructure Resilience: Maintains strict performance curves during simulated stress test profiles without manual platform staging.
